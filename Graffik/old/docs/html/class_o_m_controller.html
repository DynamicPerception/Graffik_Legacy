Detailed Description

An OMController provides the ability to control motion axes connected to a specific serial port. You must first create an OMController before attempting to create any OMAxis objects.

After you have created the controller, you may perform the actual connection to the serial port at any time, using the connect() method.

If you create an OMAxis object attached to and OMController and begin issuing commands through the OMAxis object before calling the connect() method, your commands will be queued and issued immediately upon connection.

The following provides an example of creating and using an OMController object:

  #include <QDebug>

  ...

  #include "omcontroller.h"

  ...

  int main(int argc, char *argv[])
  {

    ...

    char* serPort[] = "COM5";

    OMController* ctrl = new OMController(serPort);

    try {

        ctrl->connect();
    }
    catch (int e) {
        if( e == OpenMoCo::errSerialNotAvailable ) {
            qDebug() << "ERROR: Could not connect to serial port";
        }
        else {

            qDebug() << "ERROR: Caught signal: " << e;
        }

        exit();
    }

    qDebug() << "Successfully connected to " << serPort;

  }
